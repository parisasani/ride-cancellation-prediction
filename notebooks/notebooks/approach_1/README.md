# Approach 1 (Baseline Model)

This approach represents the initial analytical pipeline developed for predicting ride cancellations.

## Objective
The goal of this approach is to build a baseline model to understand the relationship between operational and booking-related variables and ride cancellations.

## Methodology
This approach follows a standard data analytics workflow:

1. Data Cleaning  
The dataset is preprocessed by handling missing values and preparing variables for analysis.

2. Exploratory Data Analysis (EDA)  
EDA is conducted to identify patterns and relationships between ride cancellations and key variables such as waiting time, ride distance, vehicle type, and booking features.

3. Modeling  
Basic machine learning models are implemented to predict ride cancellations.

## Description
This approach focuses on building a simple and interpretable baseline model without applying advanced feature engineering techniques.  
It helps establish a reference point for evaluating improvements in the second approach.
